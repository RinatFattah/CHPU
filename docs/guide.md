# Симуляция собственного G-кода в NX по заготовке из STEP — полный гайд (v2)

Проверено на: NX 2406, станок-пример sim01_mill_3ax (стойка Sinumerik, CSE-драйвер), G-код из FreeCAD (постпроцессор grbl_post). Гайд включает все ошибки, на которые мы наткнулись, и их решения.

**Дано:** STEP-файл заготовки (произвольной формы, например уголок) + готовый файл G-кода.
**Цель:** увидеть в NX, что именно вырежется, и сохранить результат файлом.

> ⚠️ **ЭТОТ ПРОЦЕСС УЖЕ АВТОМАТИЗИРОВАН** — `python run_cam.py деталь.step --simulate`
> (модули `nx_sim.py` / `nx_sim_journal.py` / `nx_sim_export_journal.py`). Ручной
> процесс ниже оставлен как справка и как источник «граблей». Ключевое отличие
> автоматики от ручного гайда: **захват результата НЕ через кнопку «Создать
> фасетное тело для ЗвПО»** (у неё нет API, а лента NX не видна UI Automation —
> проверено), **а через `SimulationOptionsBuilder.SaveAsPartfile = True`**: ISV
> сам сохраняет вырезанный IPW отдельным `*_ipw.prt`, который затем экспортируется
> в STEP. Запуск — через `ugraf -auto` (в чистом `run_journal` движок CSE не
> исполняет программу; нужен живой цикл GUI). Нужна лицензия `ug_isv_full`.

## Часть 0. Что важно понять до начала

1. NX **не умеет** превращать G-код обратно в операции — «вставить» его в Навигатор операций нельзя. Единственный путь — **Симуляция станка на основе кода** (модуль ISV): виртуальная стойка исполняет текст программы.
2. Для этого нужен станок с кинематикой и драйвером стойки. Если своего machine kit нет — берётся **пример из библиотеки NX** (sim01…sim0N). Они лежат в `<папка установки NX>\MACH\resource\library\machine\`.
3. CAMotics и подобные простые симуляторы для заготовки сложной формы не годятся — они умеют только прямоугольный блок.
4. **Порядок сборки проекта имеет значение:** сначала геометрия (импорт, MCS, WORKPIECE), потом извлечение станка. Сборка симуляции строится при извлечении станка и читает WORKPIECE в этот момент.

## Часть 1. Подготовка проекта

1. Создайте новый .prt: **Файл → Импорт → STEP** → файл заготовки. Сохраните.
2. **КРИТИЧЕСКАЯ ПРОВЕРКА — солид или поверхности.** Наведите курсор на импортированное тело и читайте строку состояния (самая нижняя полоска окна NX): должно быть **«Твёрдое тело»**. Вторая проверка: панель фильтра выбора сверху → поставить фильтр «Твёрдое тело» → тело должно выбираться кликом.
   - Признаки, что импортировались **поверхности (листовые тела)**: тело выглядит полым/плёночным, сквозь стенки видны рёбра; при наведении строка состояния пишет «Грань из …» / «Листовое тело»; имя тела вида «Open CASCADE STEP translator …» (файл прошёл через FreeCAD).
   - **Симуляция стойки принимает только солид.** Листовые тела WORKPIECE даст выбрать без ошибок (селектор всеяден), но стойка их молча отбракует → «No part and blank geometry has been specified».
   - **Лечение:** Меню → Вставить → Комбинировать → **Сшить (Sew)** → выбрать все поверхности тела → ОК → проверить, что стало «Твёрдое тело». Либо пере-импортировать STEP с включённой опцией сшивания в настройках импорта.
3. Перейдите в обработку: **Файл → Все приложения → Обработка** (Ctrl+Alt+M), конфигурация mill_planar.
4. **MCS (ноль программы).** Навигатор операций → вид геометрии → двойной клик MCS. Ноль должен стоять относительно заготовки **точно так же, как при генерации G-кода**. В шапке кода FreeCAD пишет систему координат: `Part: 77 x 40.1 x 21, X0 Y0 Z0 = corner-top`, `Stock box: X 0..90, Y -20..60, Z -21..24` — по этим строкам восстанавливается положение нуля.
5. **WORKPIECE.** Двойной клик WORKPIECE (под MCS):
   - **«Задать деталь»** → тело заготовки (да, то же тело — стойке нужна именно PART-геометрия: «at least a part geometry must be specified»; если есть настоящая модель детали — её).
   - **«Задать заготовку»** → тело заготовки.
   - Проверка глазом 👁 рядом с каждым полем: подсвечивается нужное тело. Счётчики (1)/(1).

**Грабли:**
- Тело может лежать на выключенном слое или быть скрытым: Ctrl+L, Ctrl+Shift+U, Навигатор сборки. Тело со скрытым/прозрачным отображением (Ctrl+J → Прозрачность) в WORKPIECE выбирается, но выглядит «плёнкой».
- Диалог «Геометрия заготовки», который выскакивает в симуляции при включении «Удаления материала», предлагает производные от ДЕТАЛИ (Ограничивающий блок, Контур детали…) — если детали нет, блок строится нулевым «сбоку» у нуля координат. Основное назначение геометрии — через WORKPIECE.

## Часть 2. Станок из библиотеки

1. Навигатор операций → **вид станков** (кнопки-переключатели видов — в верхней части самого Навигатора операций, над колонкой «Заголовок»; наводите мышь — всплывают подсказки). В этом виде дерево: станок → SPINDLE → TOOL_MAGAZINE → POCKET_01…08.
2. Двойной клик по строке станка → **«Вызвать станок из библиотеки»** → 3-осевой вертикально-фрезерный. У sim01_mill_3ax стойка **Sinumerik** (драйвер в `cse_driver\sinumerik`, программы ждёт в .MPF).
3. **Позиционирование детали** («Крепление детали»): триадой поставить заготовку на зеркало стола. При добавлении первого компонента NX спросит про **фиксированное сопряжение** — отвечайте **«Нет»** (свободный компонент проще двигать при подгонке под ноль кода).
4. Если заготовка «вросла» в стол или её надо двигать позже: **Ctrl+T (Изменить → Переместить объект)** для тела; «Сборки → Переместить компонент» для компонента. После ЛЮБОГО перемещения тела — переоткрыть WORKPIECE и проверить/переназначить деталь и заготовку, а также положение MCS.

## Часть 3. Инструмент

1. **Создание инструмента** (вкладка Исходная) → MILL/BALL_MILL → диаметр и длина реальной фрезы.
2. Раздел **«Номера»**: Номер инструмента = номер T из G-кода, Регистр настройки = тот же, Регистр коррекции = тот же.
3. Перетащить инструмент в **POCKET_01** (вид станков). Драйвер смены (ToolChange.SPF) берёт 3D-тело инструмента grasp'ом **из кармана по номеру** — в SPINDLE напрямую класть нельзя.

**Грабли «невидимый инструмент» (финальный босс первой сессии):** тело фрезы существует (подсвечивается при выделении), но не отображается без выделения и **не участвует в съёме** — симуляция едет без ошибок, Z правильный, резки нет. Проверки: Ctrl+J (прозрачность), Ctrl+L (слой), Ctrl+Shift+K. Не помогло — **удалить инструмент и создать заново** (свежий по умолчанию видим), при необходимости переизвлечь станок. Контроль: инструмент виден в графике **без выделения**.

## Часть 4. Подготовка G-кода под стойку Sinumerik

1. **Комментарии в скобках `(...)` → Parse error.** Массовая чистка в VS Code/Notepad++ (режим Regex): Найти `^;?\(.*\n` → Заменить на пусто. Либо `^\(` → `;(`.
2. **Смена инструмента — строго две строки, T перед M6:**
   ```
   T1
   M6
   ```
   Обратный порядок (`M6 T1`) → «Tool 1 not defined».
3. **G21 удалить** (Sinumerik не знает).
4. G17, G90, G54, M3 S…, G0/G1/G2/G3 X Y Z F — как есть.
5. Сохранить как **.mpf** (или в NX ставить фильтр «Все файлы»).

## Часть 5. Данные инструментов стойки (TO_INI)

Симптомы без этого шага: «Program name 'TO_INI' not found» при старте, «Error: Tool 1 not defined in tool data table. Check $TC_TP1 variable» при T1/M6.

Main.ini стойки (`...\sim01_mill_3ax\cse_driver\sinumerik\sim01_mill_3ax-Main.ini`) вызывает подпрограмму TO_INI, которой в поставке нет:

1. В подпапке **subprog** создать **TO_INI.SPF** (папка в Program Files — сохранить в другое место, скопировать с правами администратора).
2. Содержимое для T1 Ø12×75:
   ```
   $TC_TP1[1]=1
   $TC_TP2[1]="MILL_D12"
   $TC_DP1[1,1]=120
   $TC_DP2[1,1]=1
   $TC_DP3[1,1]=75.0
   $TC_DP6[1,1]=6.0
   M17
   ```
   TP1/TP2 — номер и имя; DP1=120 — тип «концевая фреза»; DP3 — длина; DP6 — радиус. Другие инструменты — те же строки с индексом [2], [3]…
3. Без этих данных стойка не активирует компенсацию длины: шпиндель едет с «нулевым» вылетом (корпус «лижет» деталь там, где должен быть кончик фрезы), тело инструмента не отрисовывается.

## Часть 6. Запуск симуляции

1. Вид программ → выделить **NC_PROGRAM** (иначе «операции для симуляции не выбраны») → правой кнопкой → **Симуляция станка**.
2. **«Симуляция внешней программы»** → «Канал Main» → кнопка с папкой → ваш .mpf. Кнопки группы ЗвПО (включая «Удаление материала») оживают только **после назначения программы**.
3. **Сменить программу:** «Диспетчер программ» (лента, группа «Виды») → правой кнопкой по назначенной программе → заменить/удалить. Там же справа виден текст программы — проверка, что читается актуальная версия.
4. **Кэш:** правки файла подхватываются после полного перезахода или замены в Диспетчере. Надёжный приём — новое имя файла (…_v2.mpf).
5. Включить **«Удаление материала»** до Play. Признак работы: заготовка сразу перефасетилась в IPW (по умолчанию голубой полупрозрачный).
6. **Настройки симуляции** → «Удаление материала»: Прозрачность **0**, Разрешение **Средний/Точный**, галки «Скрыть геометрию заготовки/детали при отображении ЗвПО» — вкл.
7. **Play**; быстрый результат — кнопка перемотки в конец (программа исполняется без анимации).
8. «Превышение предела: ось Z» — ходы примерного станка меньше реального; «Продолжить», на резку не влияет.
9. Удобный вид: **«Инструмент/ЗвПО»** — только инструмент и заготовка, станок скрыт. Прозрачность станка: диалог станка → Machine Translucency.

## Часть 6а. Ошибка Blank Geometry («No part and blank geometry has been specified»)

Формулировка врёт: геометрия может быть «указана», но **непригодна** или **не дошла до стойки**. Причины по частоте:

1. **В WORKPIECE не заполнена ДЕТАЛЬ.** Сообщение прямо требует part. Назначить в «Задать деталь» то же тело заготовки (трюк деталь = заготовка), если модели детали нет.
2. **Тело — листовое, не солид** (Часть 1, шаг 2). WORKPIECE его примет, стойка отбракует. Сшить (Sew) или пере-импортировать.
3. **Правки WORKPIECE сделаны после входа в симуляцию.** Назначения читаются при входе. Сохранить → «Завершить симуляцию» → войти заново.
4. **K-компоненты PART/BLANK станка не заполнились из WORKPIECE** (первая версия сообщения так и пишет: «Следующие К-компоненты не имеют геометрии: PART… измените К-компонент в навигаторе станка»). Лечение — назначить руками:
   - **Навигатор станка** открывается иконкой в **боковой панели** (вертикальная колонка вкладок сбоку графического окна — там же, где Навигатор операций и Навигатор сборки).
   - В дереве узлов станка найти **PART** и **BLANK** → правой кнопкой → **Изменить K-компонент** → выбор геометрии → тело заготовки → ОК.
   - Альтернатива: переизвлечь станок из библиотеки ПОСЛЕ заполнения WORKPIECE — сборка симуляции пересоберётся и подтянет геометрию.
5. Окно «Компоненты станка» на ленте симуляции — это **только видимость** узлов (галки), K-компоненты через него не назначаются.

Проверка успеха любого пункта: при включении «Удаления материала» заготовка перефасетилась в IPW, кнопка осталась нажатой.

## Часть 7. Диагностика «фреза едет, а резки нет»

Проверять по порядку, каждая проверка бинарная:

1. **Тестовый код** с заведомым врезом (координаты — по вашей заготовке):
   ```
   T1
   M6
   G17 G90 G54
   M3 S5000
   G0 X40 Y-20 Z10
   G1 Z-5 F300
   G1 X10 F500
   G0 Z50
   M30
   ```
   Канавка есть → цепочка работает, проблема в координатах большой программы. Нет → съём мёртв, дальше по списку.
2. **Инструмент виден без выделения?** Нет → пересоздать (Часть 3).
3. **Смена отрабатывает?** При M6 фреза переезжает из кармана в шпиндель? Нет → инструмент в POCKET_01, номера, TO_INI, порядок T1→M6.
4. **IPW совпадает с заготовкой у триады MCS?** Нет → в WORKPIECE старое/другое тело (после перемещений тела назначение не обновляется само) — переназначить.
5. **Высота:** пауза на рабочем ходу → «Состояние обработки» (лента, группа «Виды» — координаты осей) + Анализ → Измерить от носа шпинделя до заготовки: зазор ≈ длине фрезы.
6. **Координаты XY:** резы смещены → измерить смещение по кромкам факта, сдвинуть тело (Ctrl+T). Зеркально не туда → ориентация осей, разворот тела 180° вокруг Z в точке MCS.

## Часть 8. Сохранение результата

1. После прогона (не сбрасывая машину!) — **«Создать фасетное тело для ЗвПО»** (лента, группа ЗвПО). В файле появится фасетное тело — слепок обработанной заготовки, остаётся после «Завершить симуляцию».
2. **.prt:** Файл → Сохранить — слепок уже внутри.
3. **STEP AP242:**
   - Файл → Экспорт → STEP → протокол **обязательно AP242** (фасетные тела умеет только он; в AP203/214 не выгрузится).
   - «Экспорт из»: **Выбранные объекты** → фасетное тело (при наведении — «Фасетное тело»). Либо скрыть лишнее (Ctrl+B) и «Отображаемая деталь».
   - Путь по умолчанию смотрит в ProgramData — поменять. Проверить лог экспорта.
4. **STL** для печати/просмотрщиков: Файл → Экспорт → STL → то же тело.
5. Промежуточные состояния: жать «Создать фасетное тело для ЗвПО» на паузах в нужные моменты программы.

## Быстрый чек-лист перед каждым прогоном

- Тело заготовки — **солид** («Твёрдое тело» в строке состояния; фильтром «Твёрдое тело» выбирается).
- WORKPIECE: **и деталь, и заготовка** назначены на актуальное тело (после перемещений — переназначить).
- MCS соответствует нулю G-кода (шапка файла).
- Порядок сборки: геометрия и WORKPIECE — до извлечения станка; иначе PART/BLANK назначить руками через Навигатор станка (боковая панель).
- Инструмент: виден без выделения, в POCKET_01, номера = T кода.
- TO_INI.SPF в subprog; в коде T1 → M6, комментарии вычищены, G21 удалён.
- «Удаление материала» включено (после назначения программы), прозрачность 0, разрешение ≥ средний.
- Правленый код — перезаход или замена в Диспетчере программ.
