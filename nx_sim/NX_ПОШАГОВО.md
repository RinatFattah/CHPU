# Симуляция уголка в NX 2406 — полностью с нуля

Станок: **sim01_mill_3ax_sinumerik**. Файлы в этой папке:
`sim_stock.step` (заготовка), `sim_part.step` (деталь), `program.mpf` (G-код).

Главное правило (из-за него была половина проблем): **сначала геометрия + WORKPIECE, потом станок.**
Тогда узлы PART/BLANK станка заполнятся геометрией сами.

---

## Перед NX — сгенерировать программу и геометрию (терминал)

Три файла выше (заготовка, деталь, программа) не рисуются руками — их выдаёт пайплайн
из папки `ai_3d_pipeline`. Две команды, до захода в NX:

**1. Деталь + заготовка → управляющая программа и геометрия под симуляцию:**
```
python run_cam.py "<деталь>.stp" "out.gcode" --config config.yaml --stock "<заготовка>.stp" --nx-export
```
Строит траекторию по модели (черновая + чистовая) → `out.gcode`. `--nx-export` дополнительно
кладёт рядом `out_part.step` (деталь) и `out_stock.step` (заготовку), пересчитанные в систему
координат G-кода — после авто-ориентации и переноса нуля. Их импортируешь в Фазе 1, и они
встают точно под траекторию. Имя `out.gcode` вторым аргументом задаёт общий префикс всех
выходных файлов.

**2. Программа в формат стойки (grbl → Sinumerik `.mpf`):**
```
python grbl_to_sinumerik.py "out.gcode" "out.mpf"
```
Срезает `(...)`-комментарии, включает живую смену инструмента `T1/M6`, убирает `G21` — без
этого симуляция Sinumerik не проходит. → `out.mpf`, его грузишь в Фазе 5.

Итог — три входа для NX: деталь `out_part.step`, заготовка `out_stock.step`, программа
`out.mpf`. В этой папке они лежат под именами `sim_part.step`, `sim_stock.step`, `program.mpf` —
переименуй в них или подставляй свои пути в фазах ниже.

## Фаза 1 — Новый файл с ОБОИМИ телами

1. Прошлые эксперименты (`sim_stock_step.prt`, `sim_part_step.prt`) закрой, можно не сохранять.
2. **Файл → Создать → Модель**, единицы **мм**, назови напр. `cam_sim.prt`.
3. **Файл → Импорт → STEP → `sim_stock.step`** → OK  *(заготовка)*.
4. **Файл → Импорт → STEP → `sim_part.step`** → OK  *(деталь; ляжет ровно внутрь заготовки — координаты совпадают)*.
5. Наведись на каждое тело → в строке состояния должно быть **«Твёрдое тело»**. Если «листовое» → **Вставить → Комбинировать → Сшить**.

Теперь в одном файле два тела: крупная **заготовка** (~90×80×45) и тонкий **уголок-деталь** (~77×40×21).

---

## Фаза 2 — Обработка: MCS и WORKPIECE (ДО станка!)

1. **Ctrl+Alt+M** → приложение **Обработка** → конфигурация **mill_planar**.
2. Навигатор операций → **вид геометрии** (кнопки-переключатели вверху навигатора) → 2× клик **MCS** → ноль оставь в **абсолютном (0, 0, 0), Z вверх** → OK.
3. 2× клик **WORKPIECE** (под MCS):
   - «**Задать деталь**» → кнопка выбора → кликни **тонкий уголок** → (1);
   - «**Задать заготовку**» → кликни **крупное тело** → (1);
   - OK.

*Отличить тела: заготовка крупнее/боксовее, деталь тоньше. Наведение показывает имя тела внизу.*

---

## Фаза 3 — Станок из библиотеки (ПОСЛЕ WORKPIECE)

1. Навигатор операций → **вид станков** → 2× клик по станку → «**Вызвать станок из библиотеки**».
2. Класс **MACHINE → MILL** → выбери **`sim01_mill_3ax_sinumerik`** (столбец «Управление» = **Sinumerik**!) → OK.
3. «Добавить деталь обработки» → «**Выбрать деталь**» → кликни тело (заготовку) → привязка **Абсолютно** → **Применить**.
4. «Создать фиксированное сопряжение?» → **НЕТ**.
5. Окно «Добавить деталь обработки» всплывёт снова → **ОТМЕНА** (больше деталей не нужно).
6. Так как WORKPIECE задан ДО станка — узлы **PART/BLANK** станка должны заполниться сами.
   Проверить: **Навигатор станка** (боковая панель) → раскрой до `SETUP` → у `PART` и `BLANK` есть геометрия?
   Если пусто → ПКМ → «Изменить K-компонент» → **PART = уголок**, **BLANK = заготовка**
   (перекрытое станком тело бери через **QuickPick**: навёл, подождал секунду, клик, выбрал «Твёрдое тело»).
7. Деталь «проваливается» в стол — это косметика нашего нуля (Z0 = верх детали). **На рез не влияет**, столкновение со столом не критично. Хочешь красиво — «Сборки → Переместить компонент» → сдвинь по **+ZM**, чтоб низ лёг на стол.

## Фаза 4 — Инструмент

1. Вкладка Исходная → **Создать инструмент → MILL** → диаметр **12**, длина **75**.
2. Раскрой секцию «**Номера**» → **Номер инструмента = 1** (регистры = 1) → OK.
3. Вид станков → раскрой **[+] TOOL_MAGAZINE** → перетащи **MILL** в **POCKET_01**.

---

## Фаза 5 — Запуск симуляции по коду

1. **Поиск (Ctrl+Space)** вверху справа → набери «**Симуляция станка**» → запусти (или ПКМ по `NC_PROGRAM`).
2. Если ошибка «Операции не выбраны» — кнопка «**Симуляция траектории ▼**» → выбери «**Симуляция внешней программы**».
3. Канал **Main** → кнопка папки → `nx_sim\program.mpf` (фильтр файлов → «**Все файлы**»).
4. Группа **ЗвПО** → «**Удаление материала**» ВКЛ. Настройки: прозрачность **0**, разрешение средний/точный.
5. **Воспроизвести** (или перемотка в конец).
   ✅ Заготовка перефасетится в голубой IPW, фреза выгрызает её к форме уголка.

> На этом станке **двигается СТОЛ (X/Y), шпиндель — только Z**. «Едет стол, а не фреза» — это НОРМА, так устроен sim01_mill_3ax (в CAMotics двигался инструмент — тут наоборот, относительное движение то же).
>
> **Хочешь «деталь на месте, фреза ездит вокруг»:** в окне симуляции → **Настройки симуляции → вкладка «Отобразить» → «Стационарный» → выбрать деталь**. Тогда деталь и стол неподвижны, а движется инструмент. Съём материала и IPW — те же, меняется только «кто едет».

---

## Фаза 6 — Сохранить результат

1. Не сбрасывая машину → лента, группа **ЗвПО** → «**Создать фасетное тело для ЗвПО**».
2. **Завершить симуляцию**.
3. **Файл → Экспорт → STEP** → протокол **AP242** → «Экспорт из: Выбранные объекты» → кликни фасетное тело → задай путь → OK. *(или STL)*

## Фаза 7 — Верификация по припуску (опционально)

Проверить, насколько деталь после симуляции отличается от исходной и укладывается ли
в припуск — скриптом `verify.py`, без NX.

1. На шаге 1 «Перед NX» добавь к команде флаг **`--verify-export`** — рядом с G-кодом
   лягут `out_part.step` (эталон) и маски `out_reachable.step` / `out_unreachable.step`.
2. Результат из Фазы 6 у тебя уже в **STEP** (AP242) — им и пользуйся, напр.
   `machined.step` (STL тоже принимается).
3. В терминале:
   ```
   python verify.py machined.step --from-export out --allowance 0.5
   ```
4. Вердикт: **зарез** ниже номинала должен быть ≈ 0, **избыток** над номиналом ≤ припуска
   (на достижимых поверхностях; зоны второго установа исключаются по маске). Файл
   `machined_deviation.ply` — цветная карта (MeshLab/FreeCAD): красный = избыток сверх
   припуска, синий = зарез, серый = второй установ. Код возврата 0 (PASS) / 2 (FAIL).

Нужны пакеты `numpy`/`scipy`/`trimesh` (`pip install -r requirements.txt`). Оба входа —
в СК G-кода: эталон её несёт (`--verify-export`), симуляция гоняет ту же программу.

## Если на Фазе 5 всё равно «чушь» (не режет / не сходится)

Проверь по порядку:
- **BLANK = заготовка** (крупное тело), а не деталь/пусто — иначе резать нечего (частая причина).
- **TO_INI.SPF** на месте (Фаза 0); в программе `T1`/`M6`; фреза в `POCKET_01`, номер = 1.
- Ноль/крепление сошлись по Z.

Позиционирование/крепление — та самая «не всё чисто» зона из гайда Рината. Если после проверок не сходится — скинь Ринату скрин + «стол едет, деталь утоплена, съёма нет»: он это проходил и быстро скажет точный трюк.
