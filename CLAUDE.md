# CLAUDE.md

## Project Summary

CAM pipeline: **CAD model ‚Üí G-Code** for a 3-axis CNC mill, generated by FreeCAD's
Path/CAM workbench (headless). No toolpath is supplied ‚Äî it is computed from the model.
Default output is **staged roughing** ("like a machinist"), ROUGH_MODE=stages, in
this order: RoughHole1..N FIRST (through-cutouts of ARBITRARY shape = inner wires of
the part silhouette, nearest-first, Adaptive ‚Äî cut while the part is still rigidly
held in the stock) ‚Üí faces top-down, planar and slanted INTERLEAVED by height, taking
ONLY faces reachable from above (stock above the face, upward normal = nonzero XY
projection, and NO part body above it ‚Äî not covered, not an overhang): planar
(shelves/pocket floors/steps) as RoughFace1..N via Adaptive, slanted/curved (chamfer,
dome, bend radius) as RoughSlope1..N via terraced Path Surface (Multi-pass +
DepthOffset=allowance; the face is usually NARROWER than the tool, so CutPattern=ZigZag
+ BoundaryAdjustment=tool radius + BoundaryEnforcement=False are REQUIRED ‚Äî the default
Offset pattern insets by the tool radius and yields an empty path) ‚Üí RoughPerimeter
LAST: a single Path Profile pass (Side=Outside, offset radius+allowance) around the
part's FILLED silhouette (outer wires only, via Part::FaceMakerBullseye). The outer
contour is cut last so the part stays rigid in the stock frame during all internal
work; ONLY the part perimeter is cut ‚Äî leftover stock in the corners that does not
touch the part is intentionally LEFT (this mode does NOT bulk-clear the whole margin,
so a single-pass perimeter into a wide margin is a buried full-width slot). Holes and
planar faces are Adaptive ops over explicit non-overlapping 2D regions. Allowance is
left per `ROUGH_ALLOWANCE_MODE` (none = clean, no allowance = DEFAULT; xy = walls only;
all = walls + floors; magnitude `ROUGH_ALLOWANCE`): walls via StockToLeave / Profile
OffsetExtra, floors via FinalDepth ‚Äî EXCEPT through-holes (RoughHole) and the outer
contour (RoughPerimeter),
cut to the PART bottom (bb.ZMin, no floor allowance): a through-hole has no floor to
finish and the perimeter must part the piece from the frame, so leaving allowance there
strands a skin at the bottom (visible with FINISH off). Down/side-facing or covered faces are skipped (warned, second setup);
material unreachable from above (overhangs) is skipped ‚Äî
that's a second setup, not something to cut through. Silhouette sections MUST be
built with `Part.makeFace(wires, "Part::FaceMakerBullseye")` ‚Äî making a Face per
wire fills cutouts and the union loses the holes. A Path Surface finishing
pass is OFF by default (`--finish` / `FINISH: true` enables it). Stock = part
bbox + margins (`STOCK_MARGIN`, default 12 = 2 tool √ò) OR an arbitrary solid from a
file (`STOCK_FILE` / `--stock`): the stock file must be in the SAME coordinate
system as the model ‚Äî the worker records the part's orient/origin transforms in a
journal and replays them on the stock. When one stock file serves a whole part
family (its coords match only ONE part), `STOCK_ALIGN` / `--stock-align` ignores the
file's coordinates: the stock is laid flat, edges axis-aligned, then placed
corner-in-corner (X min-flush, Y centered, Z bottom-to-bottom ‚Äî surplus on top where
the tool can reach it). Part orientation beyond auto_orient: orient_hole_axis_up
(dominant ‚â•180¬∞ concave-cylinder axis ‚Üí vertical; quarter-cylinder bend fillets are
excluded on purpose), orient_flange_down (largest horizontal face to the bottom half,
else the wall hangs as an overhang), orient_wall_to_yz (largest vertical planar face's
normal ‚Üí X). Narrow through-slots where Adaptive can't helix (slot > tool √ò but
< 2 √ò) fall back to Path Profile Side=Inside (plunge entry at VertFeed). FreeCAD 1.1 has no `Stock.CreateFromExisting`,
but assigning a plain `Part::Feature` to `job.Stock` works (ops only read
`job.Stock.Shape`); the contour zone is then built from the stock's real silhouette.
An INVALID stock solid silently breaks Adaptive (empty paths) ‚Äî the worker attempts
a sew+makeSolid repair and warns. Stock/part/tool dimensions are
embedded as `(Stock: ...)`/`(Part: ...)`/`(Tool: ...)` header comments in the
G-Code. The stock in PROGRAM coordinates (rotated/shifted with the part) is
exported next to the G-Code as `<gcode_stem>_stock.stp` (worker `export_stock`;
written via ASCII temp path because OCCT can't write non-ASCII paths on Windows).
The customer's source parts are Siemens NX `.prt`; FreeCAD cannot read `.prt` (no
open-source readers exist). On machines with NX installed, `run_cam.py` accepts
`.prt` directly (model and `STOCK_FILE`): `nx_export.py` converts it to STEP
(AP242 by default, `NX_STEP_AP`) via NX's command-line translator
`<NX>\TRANSLATORS\step242\step242.exe input.prt o=out.stp d=ugstep242.def l=log`
‚Äî the translator needs NXBIN prepended to PATH (else 0xC0000135) and ROSE/ROSE_DB
env vars pointing at its folder. Do NOT use the NXOpen DexManager.StepCreator
route headless: ObjectTypes default to all-False (SettingsFile does not override
them) and even with them set the headless export yields a geometry-less STEP
(views/cameras only). Without NX, the bridge remains manual STEP export from NX.

CRITICAL Adaptive gotcha: regions must be explicit planar faces. Passing model faces
as `Base` yields open projected contours on real parts (`Path.Area: ccurve not closed`)
and Adaptive silently returns an empty path. Adaptive also needs ~2 tool diameters of
material width for its helix entry ‚Äî narrow perimeter slivers (e.g. corner rounds at
`STOCK_MARGIN` 0) get a partial/empty path, hence the Profile fallback for the contour.
Orientation gotcha: the "features up" flip must test hole OPENNESS (`isInside` probes
above/below the cylinder), not "cylinder near the bottom" ‚Äî a through-window in a
bottom plate would otherwise flip the part upside down and make everything but the
window an unreachable overhang.

An earlier LLM/Blender text‚ÜíSTL demo pipeline was removed from the repo (July 2026);
it lives in git history if ever needed.

## Files

Layout: CLI entry points (`run_cam.py`, `auto_fix.py`) + `config.py` at repo root;
FreeCAD host modules and workers in `cam/`; everything Siemens NX in `nx/`
(recorded research journals in `nx/research/`); docs in `docs/` (incl.
`docs/LLM_–ü–ï–¢–õ–Ø.md` ‚Äî full LLM-loop tech writeup); sample parts in `samples/`.
In-package modules bootstrap repo root onto sys.path, so both `python run_cam.py`
and direct runs like `python cam/step_diff.py a.step b.stp` work.

- `run_cam.py` - CLI entry point: `python run_cam.py model.step|model.prt [out.gcode] [--config config.yaml]`
- `auto_fix.py` - autonomous LLM loop: generate ‚Üí NX-simulate ‚Üí boolean diff
  (`cam/step_diff.py` + `cam/freecad_diff_worker.py`) ‚Üí ask `claude -p` (headless CLI; found
  via PATH / CLAUDE_CLI env / VSCode extension native-binary) ‚Üí parse STRICT-JSON
  actions (set_param whitelist with bounds; extra_zone = forced clearing rect for
  undercuts, Adaptive with Profile-inside fallback, floor-clamped; skip_op = drop a
  named op, validated against op-name pattern; dead_zone XY boxes) ‚Üí regenerate.
  Op names for the prompt are parsed from `(Begin|Finish operation: X)` G-code
  comments. Boolean diff gotcha: OCCT booleans with the FACETED sim body are only
  reliable with it as the RIGHT operand (result.cut(part) silently no-ops) ‚Äî hence
  undercut = (bbox‚àípart) ‚àí ((bbox‚àípart) ‚àí result), the sim body is mesh-laundered
  (tessellate‚ÜímakeShapeFromMesh‚ÜímakeSolid, reverse if inverted), the intentional
  floor skin is split off by a slab cut, and diffuse tessellation films
  (fill<2% of own bbox) are filtered as noise.
  Journal at `<gcode>_autofix.json`. Key safety: `FLOOR_CLEARANCE` (default 0.5 mm)
  clamps FinalDepth of EVERY op (incl. through-holes/perimeter/finish) to part
  bottom + clearance ‚Äî the part lies on the machine table; `DEAD_ZONES` (XY boxes,
  program coords) are subtracted from hole/face regions, slopes with center inside
  are skipped (perimeter Profile is NOT affected). Diff counts undercut only inside
  the part-silhouette prism (stock frame is intentional), bottom skin ‚â§ floor limit
  reported separately as floor_skin, faceting noise filtered by min_volume.
- `nx/nx_export.py` - Siemens NX bridge: `.prt` ‚Üí STEP via NX's CLI translator (headless).
- `cam/freecad_cam.py` - host side: locates `freecadcmd`, passes params via temp JSON
  (env var `FREECAD_WORKER_PARAMS`), parses `[worker]` output markers.
- `cam/freecad_worker.py` - runs **inside** FreeCAD's interpreter: model ‚Üí solid ‚Üí
  origin normalization ‚Üí Path Job ‚Üí Surface op ‚Üí postprocessor ‚Üí G-Code.
- `nx/nx_sim.py` + `nx/nx_sim_journal.py` + `nx/nx_sim_export_journal.py` - NX ISV
  material-removal simulation of the G-code (`--simulate`); see the ISV Key Fact.
- `cam/step_describe.py` + `cam/freecad_describe_worker.py` - CAD file ‚Üí compact JSON
  geometry description for LLM/automation (bbox, faces by type, holes, summary).
- `nx/grbl_to_sinumerik.py` - standalone GRBL‚Üí.mpf converter (manual NX sim path,
  see `docs/NX_–ü–û–®–ê–ì–û–í–û.md`).
- `config.py` - parameter defaults + YAML loading (`--config`).
- `README_CAM.md` - operator-facing parameter reference. **Keep in sync** when
  changing CAM params. `README.md` - main handoff doc.

## Key Facts

- Input formats: STEP/IGES/BREP read as exact BREP solids (units from file,
  `STL_SCALE_TO_MM` ignored); STL/OBJ via mesh‚Üífaceted-solid conversion; `.prt`
  auto-converts via NX when installed (`nx_export.py`), otherwise fails with a
  clear message (both in `run_cam.py` and the worker).
- NX on this machine: NX 2406 at `C:\Program Files\Siemens\NX2406`
  (`UGII_BASE_DIR` set globally, license via `SPLM_LICENSE_SERVER`).
  `NXBIN\run_journal.exe` runs NXOpen journals headless (works, takes a license).
- NX ISV G-code simulation IS implemented (`--simulate` / `SIMULATE`; `nx_sim.py`
  + `nx_sim_journal.py` + `nx_sim_export_journal.py`). Runs external `.mpf`
  through a virtual Sinumerik controller (CSE) with material removal on a
  library machine (`NX_SIM_MACHINE`, default `sim01_mill_3ax_sinumerik`), and
  captures the cut in-process workpiece as `<gcode>_sim.stp`. Hard-won facts:
  (1) the "Create IPW Facet Body" ribbon button (`UG_CAM_ISV_EXPORT_IPW`) has
  NO NXOpen API ‚Äî confirmed by two recorded journals (both show it as a bare
  comment with zero API calls) ‚Äî and NX's Qt ribbon is invisible to Windows UI
  Automation, so neither an API call nor a synthetic click works. The working
  path is `SimulationOptionsBuilder.SaveAsPartfile = True` (license
  `ug_isv_full`): ISV then auto-saves the cut IPW as a separate `<stem>_N_ipw.prt`
  which is exported to STEP AP242 ED2 (facet body) by a batch journal.
  (2) CSE does NOT execute under headless `run_journal` ‚Äî it needs a live GUI
  event loop; launch via `ugraf.exe -auto=<journal>` and pump the Windows message
  queue during PlayForward. (3) The `.mpf` must be cleaned for Sinumerik: strip
  `(...)` comments accounting for NESTED parens (FreeCAD's `(Tool: endmill (flat)‚Ä¶)`
  header otherwise leaves a garbage line that halts the controller at line 1),
  drop `G21`, emit `T<n>` then `M6` on separate lines, `M2`‚Üí`M30`. (4) TO_INI.SPF
  ($TC_* tool table) must exist in the machine's `cse_driver\sinumerik\subprog`.
  WORKPIECE needs BOTH part and blank set to the stock body; K-components
  PART/BLANK are assigned explicitly; MCS is left at default (stock already in
  program coords). CAMSetup.CreateFacetBodyForIpwDisplay / SinumerikCaFacetBuilder
  capture the UNCUT box (12 triangles) ‚Äî do NOT use them for external-NC IPW.
- FreeCAD runs as a **separate headless process** (`QT_QPA_PLATFORM=offscreen`);
  its Qt/OpenCASCADE never load into the main Python. Params go through a JSON file
  referenced by env var ‚Äî NOT argv (freecadcmd executes every file argument as a script).
- Worker is executed by freecadcmd as a plain script (not `__main__`) ‚Äî hence the
  `if os.environ.get("FREECAD_WORKER_PARAMS"): main()` guard at the bottom.
- `ORIGIN` normalization (default `corner-top`: X0Y0 = bbox corner, Z0 = top) exists
  because NX parts are often exported in assembly/machine coordinates, meters away
  from zero ‚Äî without the shift the first G0 sends the tool far away.
- FreeCAD on this machine (Windows 10, installed via winget):
  `C:\Users\—Ä–∏–Ω–∞—Ç\AppData\Local\Programs\FreeCAD 1.1\bin\freecadcmd.exe` (v1.1.1).
  `freecad_cam.py` auto-detects it (globs `FreeCAD*\bin\freecadcmd.exe` under
  Program Files / %LOCALAPPDATA%\Programs). The pipeline previously ran on
  Ubuntu 24.04 via extracted AppImage (apt has no FreeCAD there; snap broken
  headless) ‚Äî that path is still in the Linux candidate list.
- FreeCAD's CAM has **no lathe/turning operations** ‚Äî turning was prototyped and
  removed; don't promise it.
- Windows gotchas (both fixed in code, keep in mind when editing): (1) FreeCAD/OCCT
  cannot open files under non-ASCII paths (`C:\Users\—Ä–∏–Ω–∞—Ç\...` ‚Üí "Unknown exception
  while processing file") ‚Äî `freecad_cam.py` converts worker/model/stock paths to
  8.3 short names via `_ascii_safe()`; (2) stdout inside freecadcmd is cp1251, so a
  `print` with `√ò`/`¬≤` kills the worker with UnicodeEncodeError ‚Äî the worker
  reconfigures stdout/stderr to UTF-8 at import, and the host reads the pipe with
  `encoding="utf-8", errors="replace"`.

## Editing Guidance

- Keep documentation in Russian unless asked otherwise; `README.md` is the primary
  handoff doc, `README_CAM.md` the parameter reference.
- When changing runtime behavior or params, update `README.md`, `README_CAM.md`
  and `config.example.yaml` together.
- Generated `.stl`/`.gcode` are gitignored; keep them out of commits.
- `config.yaml` may contain operator-specific settings and is gitignored;
  the committed template is `config.example.yaml`.
- Don't describe the output as full CAM: 3-axis, single tool, no roughing/finishing
  split, no collision control ‚Äî README's "–û–≥—Ä–∞–Ω–∏—á–µ–Ω–∏—è" section is the honest contract.
