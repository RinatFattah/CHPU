# CAM: CAD-модель → G-Code (FreeCAD)

Генерация управляющей программы (G-Code) для 3-осевого фрезерного станка с ЧПУ
**по готовой CAD-модели детали**. Траектория на вход не подаётся — она строится
по модели. По умолчанию — **черновая по этапам** (вырезы → грани сверху вниз →
внешний контур детали); опционально добавляется **чистовой 3D-проход по
поверхности** (`--finish`), где фреза следует за фактической геометрией (наклоны,
конусы, купола, рельеф).

```
деталь.prt (Siemens NX) ──► python run_cam.py деталь.prt ──► деталь.gcode
                              │ 1) NX (headless): .prt → STEP AP242
                              │ 2) FreeCAD Path/CAM (headless): STEP → G-Code
деталь.step ─────────────────► сразу шаг 2
```

Siemens NX `.prt` — закрытый формат, FreeCAD его не читает. Если на машине
установлен NX, `run_cam.py` принимает `.prt` напрямую: деталь автоматически
конвертируется в STEP (AP242, настраивается `NX_STEP_AP`) штатным транслятором NX
без открытия GUI. Без NX — экспортируйте вручную: File → Export → STEP,
и подайте `.step`.

## Требования

- Python 3.10+ и `pyyaml` (`pip install -r requirements.txt`)
- **FreeCAD 1.0+** — CAM-движок (установка ниже)
- Siemens NX (опционально) — только для входа `.prt`; найдётся автоматически
  (`UGII_BASE_DIR` / `C:\Program Files\Siemens\NX*`)

## Установка FreeCAD

### Windows

Через winget (или обычный инсталлятор со страницы
[releases](https://github.com/FreeCAD/FreeCAD/releases)):

```powershell
winget install --id FreeCAD.FreeCAD -e
```

`freecadcmd.exe` ищется автоматически в `C:\Program Files\FreeCAD*\bin`,
`C:\Program Files (x86)\FreeCAD*\bin` и `%LOCALAPPDATA%\Programs\FreeCAD*\bin`
(portable-сборку `.7z` достаточно распаковать в одну из этих папок).
Другое расположение — укажите в конфиге: `FREECAD_CMD: "C:\путь\bin\freecadcmd.exe"`.

### Linux (AppImage, без root)

FreeCAD **отсутствует в репозиториях Ubuntu 24.04**, а snap-версия ломается в headless
из-за рассинхрона Qt. Рабочий способ — официальный AppImage:

```bash
mkdir -p ~/freecad-appimage && cd ~/freecad-appimage
# взять свежий Linux-x86_64 AppImage со страницы https://github.com/FreeCAD/FreeCAD/releases
wget https://github.com/FreeCAD/FreeCAD/releases/download/1.1.1/FreeCAD_1.1.1-Linux-x86_64-py311.AppImage
chmod +x FreeCAD_*.AppImage
./FreeCAD_*.AppImage --appimage-extract     # распаковка в squashfs-root/
```

`freecadcmd` окажется в `~/freecad-appimage/squashfs-root/usr/bin/freecadcmd` — этот
путь ищется автоматически. Другое расположение — укажите в конфиге: `FREECAD_CMD: "/путь"`.

### Проверка

```bash
python -c "import freecad_cam; print(freecad_cam.find_freecadcmd())"
```

## Быстрый старт

```bash
cp config.example.yaml config.yaml   # и подставить свою фрезу/подачу/стойку

python run_cam.py деталь.step --config config.yaml
# → деталь.gcode рядом с моделью
# → деталь_stock.stp — заготовка в координатах программы (повёрнутая вместе
#   с деталью): для симулятора и наладки
```

Флаги CLI: `--config FILE`, `--origin corner-top|center-top|model` (ноль программы),
для мешей — `--mm` / `--meters` / `--scale N`.

## Параметры

Все параметры генерации (инструмент, режимы резания, стратегия, ноль программы,
постпроцессор) задаются в `config.yaml`. **Полный справочник с рекомендациями —
[README_CAM.md](README_CAM.md).** Минимум, который нужно выставить под свой станок:

```yaml
TOOL_DIAMETER: 6.0        # фреза, которая реально стоит в шпинделе
FEED_RATE: 800            # подача под материал
SPINDLE_SPEED: 12000
POSTPROCESSOR: "grbl"     # диалект стойки: grbl / linuxcnc / fanuc / mach3_mach4 …
```

## Как это устроено

| Файл | Роль |
|---|---|
| `run_cam.py` | CLI: модель → G-Code |
| `auto_fix.py` | автономная ЛЛМ-петля: генерация → симуляция → diff → правка параметров через `claude -p` → регенерация (разбор: [docs/LLM_ПЕТЛЯ.md](docs/LLM_ПЕТЛЯ.md)) |
| `config.py` | дефолты параметров + загрузка YAML |
| `cam/freecad_cam.py` | хост: находит `freecadcmd`, передаёт параметры, разбирает результат |
| `cam/freecad_worker.py` | исполняется внутри FreeCAD: модель → тело → Path Job → операции → постпроцессор |
| `cam/step_describe.py` + `cam/freecad_describe_worker.py` | CAD-файл → JSON-описание геометрии (для LLM/автоматизации) |
| `cam/step_diff.py` + `cam/freecad_diff_worker.py` | булев diff «деталь vs результат симуляции» → JSON недорезов/зарезов |
| `nx/nx_export.py` | мост NX: `.prt` → STEP штатным транслятором NX (headless) |
| `nx/nx_sim.py` + журналы | симуляция G-Code на виртуальном станке NX ISV со съёмом материала (`--simulate`) |
| `nx/grbl_to_sinumerik.py` | GRBL → Sinumerik `.mpf` для ручной симуляции ([docs/NX_ПОШАГОВО.md](docs/NX_ПОШАГОВО.md)) |
| `samples/` | пробные детали и заготовка-уголок серии 75.6121 |

### CAD-файл → JSON (для LLM)

```bash
python cam/step_describe.py заготовка.stp [выход.json]
```

Сырой STEP — граф сущностей с тысячами координат, LLM его читает плохо.
Утилита сводит геометрию к компактному JSON: габарит, объём, грани по типам
с размерами и ориентацией, отверстия (Ø, глубина, сквозное/глухое) и
однострочная текстовая сводка (`summary`). Вход — STEP/IGES/BREP или `.prt`
(при установленном NX). Отверстия распознаются по вертикальной оси — файл
лучше подавать в нормализованных координатах (например, `_stock.stp` из
генерации, а не исходник в координатах сборки).

FreeCAD работает **отдельным headless-процессом** — его Qt/OpenCASCADE не грузятся
в основной Python. Ноль программы по умолчанию: X0 Y0 = угол габарита детали,
Z0 = её верхняя плоскость (`ORIGIN: corner-top`).

## Симуляция на виртуальном станке NX (опционально)

При установленном NX флаг `--simulate` (или `SIMULATE: true`) прогоняет
сгенерированный G-Code через **виртуальную стойку Sinumerik** (модуль ISV,
станок из библиотеки NX) со съёмом материала и сохраняет **обработанную
заготовку** рядом с G-Code:

```bash
python run_cam.py деталь.step --simulate
# → деталь.gcode + деталь_sim.stp (вырезанная заготовка, фасетное тело)
```

Что это даёт: проверку, что программа исполнима на реальном диалекте стойки
(машинное время, коллизии/лимиты), и фактическую геометрию результата съёма из
произвольной заготовки. Захват результата автоматический — через
`SaveAsPartfile` (нужна лицензия `ug_isv_full`); на время прогона открывается
окно NX, но взаимодействия оно не требует. Станок задаётся `NX_SIM_MACHINE`
(по умолчанию учебный `sim01_mill_3ax_sinumerik`); для «как на моём станке»
нужен machine kit конкретного станка. Подробности и грабли — в
[docs/guide.md](docs/guide.md), ручной маршрут — [docs/NX_ПОШАГОВО.md](docs/NX_ПОШАГОВО.md).

## Автономное исправление программой ЛЛМ (auto_fix)

```bash
python auto_fix.py деталь.stp --stock заготовка.stp --stock-align --iters 3
```

Петля: генерация → симуляция NX → булев diff «деталь vs вырез» (недорезы/зарезы
с координатами) → факты уходят ЛЛМ (`claude -p`, headless Claude Code, без ключей
API) → ответ-JSON с действиями применяется: `set_param` (белый список параметров
с границами), `extra_zone` (принудительная дообработка недореза по месту),
`skip_op` (отключить операцию, дающую зарез), `dead_zone` (запретная зона для
фрезы) → регенерация. Остановка:
дефекты в допуске, вердикт ЛЛМ «не лечится параметрами» (второй установ) или лимит
итераций. Журнал: `<gcode>_autofix.json`.

Безопасность: `FLOOR_CLEARANCE` (0.5 мм) — фреза никогда не опускается ниже
«дно детали + зазор», деталь лежит на столе; сквозной прорез оставляет плёнку.

## Проверка перед станком

1. Прогнать G-Code в симуляторе: `--simulate` (виртуальный станок NX, см. выше),
   либо [CAMotics](https://camotics.org) / [ncviewer.com](https://ncviewer.com)
   — траектория, глубины, зарезы.
2. Сверить `POSTPROCESSOR` с контроллером станка.
3. Выставить ноль на станке по соглашению `ORIGIN` (угол детали, Z0 = верх).
4. Контроля столкновений в самой генерации нет — пробный проход над заготовкой
   обязателен (либо прогнать `--simulate`, где ISV проверяет коллизии/лимиты).

## Ограничения

- 3 оси, обработка сверху: поднутрения, обратная сторона, боковые элементы — недоступны.
- Один инструмент на программу; черновая и чистовая не разделяются.
- Оснастка (прижимы) не моделируется; заготовка = габарит модели.
- Это автоматический генератор для типовых деталей, а не замена профессиональной
  CAM-системы (NX, T-FLEX): многоось, стратегии, контроль столкновений — там.
